# Requirements: llm-usage

*Generated from conversation on 2026-01-22*

## Objective
Build a Bash CLI tool that analyzes AI coding assistant usage statistics from OpenCode and Claude Code, outputting JSON data for the current project directory (or all projects with `--all` flag).

## Core Requirements

### Must Have (High Priority)
- **[HIGH]** Project-aware by default - analyzes current directory
- **[HIGH]** Walk up parent directories to find matching project (like git)
- **[HIGH]** Global view with `--all` flag for all projects
- **[HIGH]** JSON output to stdout (pipeable to other tools)
- **[HIGH]** Separate sections for OpenCode and Claude Code
- **[HIGH]** Session counts per model
- **[HIGH]** Message counts per model
- **[HIGH]** Token usage per model (input/output tokens)
- **[HIGH]** Usage over time (daily breakdown)
- **[HIGH]** Configurable date range: `--from YYYY-MM-DD` and `--to YYYY-MM-DD`
- **[HIGH]** Error with helpful message if no data found for project

### Should Have (Medium Priority)
- **[MEDIUM]** Error message lists nearby projects that do have data
- **[MEDIUM]** Support for `--project PATH` to specify a specific project

### Could Have (Low Priority)
- **[LOW]** Pretty-printed terminal output as alternative to JSON
- **[LOW]** Export to CSV format

## Technical Constraints
- **Language:** Bash script
- **Dependencies:** `jq` (required for JSON parsing)
- **Performance:** Should complete within 5 seconds for typical projects

## Architecture & Design
- **Pattern:** Single-purpose CLI utility (Unix philosophy)
- **Structure:** Modular functions for each data source (OpenCode, Claude Code)
- **Key Functions:**
  - `find_project_root()` - walks up directories looking for project
  - `load_opencode_stats()` - parses OpenCode JSON files
  - `load_claude_code_stats()` - parses Claude Code JSONL files
  - `aggregate_by_model()` - groups stats by model name
  - `filter_by_date_range()` - filters data by date
  - `output_json()` - prints final JSON to stdout

## User Context
**Target Users:** Developers who use multiple AI coding assistants (OpenCode, Claude Code)

**Primary Use Case:**
```bash
cd ~/dev/my-project
llm-usage
# Output: JSON stats for this project only
```

**Secondary Use Case:**
```bash
llm-usage --all
# Output: JSON stats for all projects
```

**User Flow:**
1. Navigate to project directory
2. Run `llm-usage`
3. See JSON output with usage stats
4. Pipe to `jq` for pretty printing or other tools for analysis

## Edge Cases & Considerations
- **No jq installed:** Tool should exit with clear error message and installation instructions
- **No project found:** Should error with helpful message listing nearby projects with data
- **Project found but no sessions:** Return valid JSON with zero counts
- **Date range filters:** Should handle partial ranges (only `--from` or only `--to`)
- **Timezone:** Timestamps should be handled consistently (prefer UTC for consistency)

## Success Criteria
- ✓ Tool correctly identifies project from current directory (or walks up parents)
- ✓ Accurately counts sessions and messages per model
- ✓ Correctly aggregates token usage (input/output)
- ✓ JSON output is valid and parseable
- ✓ Works with both OpenCode and Claude Code data
- ✓ Date filtering works correctly
